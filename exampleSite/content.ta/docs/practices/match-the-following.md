---
title: "பொருத்துக"
---

**பொருத்துக (Match the Following)** என்பது வெவ்வேறு கருத்துக்கள், வடிவங்கள் அல்லது கூறுகளுக்கு இடையிலான தொடர்புகளைச் சோதிக்கப் பயன்படும் ஒரு மாறும் மதிப்பீட்டுக் கருவியாகும் (dynamic assessment tool). நமது தொழில்நுட்பப் பாடப்பிரிவுகளில், குறிப்பிட்ட மென்பொருள் கருவிகளை அவற்றின் அடிப்படை வடிவமைப்புத் திட்டங்களுடன் (architectural patterns) இணைக்க அல்லது சொற்களை அவற்றின் விளக்கங்களுடன் இணைக்க இது அடிக்கடி பயன்படுத்தப்படுகிறது.

---

## "பொருத்துக" என்றால் என்ன?

இந்த வடிவத்தில், மாணவருக்கு இரண்டு பட்டியல்கள் வழங்கப்படும். **தேர்வுகள் (Choices)** பட்டியலில் உள்ள ஒவ்வொரு உருப்படியையும், **பொருத்தங்கள் (Matches)** பட்டியலில் உள்ள சரியான இணையுடன் இணைப்பதே இதன் குறிக்கோள். வகைப்படுத்தப்பட்ட அறிவு மற்றும் தொடர்புடைய தர்க்கத்தை (relational logic) மதிப்பிட இது ஒரு சிறந்த வழியாகும்.

### முக்கிய கூறுகள்:
1.  **Metadata Block:**
    * `choices`: இணைக்கப்பட வேண்டிய முதன்மை உருப்படிகள்.
    * `matches`: அவற்றுடன் பொருந்தக்கூடிய இலக்கு உருப்படிகள் (இவை கணினியால் வரிசை மாற்றப்பட்டு காட்டப்படும்).
2.  **வினா:** `##` தலைப்பைப் பயன்படுத்தி வழங்கப்படும் தெளிவான அறிவுறுத்தல்.
3.  **விளக்கம் (Rationale):** ஒவ்வொரு இணைப்பிற்குப் பின்னாலும் உள்ள தர்க்கத்தைத் தெளிவாக விளக்கும் பகுதி. இது மாணவர் "ஏன்" இந்த இணைப்புகள் செய்யப்பட்டன என்பதைப் புரிந்துகொள்ள உதவுகிறது.

---

## பயன்பாட்டு உதாரணம்: மென்பொருள் வடிவங்கள் (Software Patterns)

இந்த உதாரணம், தரநிலையான ஜாவா/புரோகிராமிங் கூறுகளுக்குள் இருக்கும் **வடிவமைப்பு வடிவங்களை (Design Patterns)** கண்டறியும் மாணவரின் திறனைச் சோதிக்கிறது.

```markdown
---
choices:
  - Collection Streams API
  - Buffered Reader
  - Container
matches:
  - Builder
  - Decorator
  - Composite
---

## பொருத்துக: புரோகிராமிங் கூறுகளை வடிவமைப்பு வடிவங்களுடன் இணைக்கவும்

#### தர்க்கம் மற்றும் விளக்கம்:
* **Collection Streams API** என்பது **Builder** உடன் பொருந்துகிறது: ஸ்டீம்ஸ் ஏபிஐ (Streams API) ஒரு வரிசையான செயல்பாடுகளை (filter, map போன்றவை) உருவாக்கி இயக்குவதற்கு பில்டர் போன்ற முறையைப் பயன்படுத்துகிறது.
* **Buffered Reader** என்பது **Decorator** உடன் பொருந்துகிறது: ஜாவா I/O-வில், `BufferedReader` ஏற்கனவே இருக்கும் ஒரு ரீடரின் இடைமுகத்தை மாற்றாமல், அதற்குத் கூடுதல் சேமிப்புத் திறனை (buffering) வழங்குகிறது.
* **Container** என்பது **Composite** உடன் பொருந்துகிறது: பயனர் இடைமுகக் கொள்கலன்கள் (Containers) தனித்தனி கூறுகளை மற்றும் கூறுகளின் குழுக்களை ஒரே மாதிரியாகக் கையாளுகின்றன. இதுவே காம்போசிட் வடிவத்தின் அடிப்படை.
```

## பயிற்சி உதாரணம்: ஆவணமாக்கல் தொகுதி (Documentation Module)
Markdown குறியீடுகளுக்கும் அவற்றின் வெளியீட்டிற்கும் இடையிலான தொடர்பைச் சோதித்தல்.

```markdown
---
choices:
  - #
  - >
  - ![]()
matches:
  - தலைப்பு (Heading)
  - மேற்கோள் தொகுதி (Blockquote)
  - படம் (Image)
---

## பொருத்துக: Markdown குறியீடுகளை அவற்றின் காட்சி கூறுகளுடன் இணைக்கவும்

#### தர்க்கம் மற்றும் விளக்கம்:
* **#** என்பது **தலைப்பு (Heading)** உடன் பொருந்துகிறது: இந்தக் குறியீடு ஆவணத்தின் கட்டமைப்பைத் தீர்மானிக்கும் தலைப்பு நிலைகளை வரையறுக்கிறது.
* **>** என்பது **மேற்கோள் தொகுதி (Blockquote)** உடன் பொருந்துகிறது: இது குறிப்புகள் அல்லது ஆலோசனைகளை மற்ற உரைப்பகுதிகளிலிருந்து தனித்துத் தெரியச் செய்யப் பயன்படுகிறது.
* **![]()** என்பது **படம் (Image)** உடன் பொருந்துகிறது: ஆச்சரியக்குறி, அடைப்புக்குறிகள் மற்றும் பிறை அடைப்புக்குறிகளின் கலவையானது படங்களை இணைப்பதற்கான தரநிலையான முறையாகும்.
```

## உள்ளடக்கத்தை உருவாக்குபவர்களுக்கான சிறந்த நடைமுறைகள்
* **சீரான இணைகள்:** பொருத்தங்கள் பட்டியலில் கூடுதல் விருப்பங்களை (distractors) வைக்க அனுமதி இருந்தாலும், மாணவரை குழப்பமடையச் செய்யாதவாறு பட்டியலைச் சுருக்கமாக வைக்கவும்.
* **ஒரே வகைப்பட்ட உருப்படிகள்:** ஒரு தொகுப்பில் உள்ள அனைத்து உருப்படிகளும் ஒரே தர்க்கத்தைச் சார்ந்ததாக இருப்பதை உறுதி செய்யவும் (உதாரணமாக: அனைத்தும் வடிவமைப்பு வடிவங்கள் அல்லது அனைத்தும் கணிதக் குறியீடுகள்).
* **தெளிவான விளக்கங்கள்:** பொருத்துக வினாக்கள் பல புள்ளிகளை உள்ளடக்கியிருப்பதால், விளக்கப் பகுதியில் ஒவ்வொரு இணைப்பையும் தனித்தனியாகப் புல்லட் புள்ளிகளில் விளக்கவும்.
