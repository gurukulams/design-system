எழுத்துரு அமைப்பு (Typography) என்பது எழுதப்பட்ட மொழியை **தெளிவாகவும், வாசிக்கக்கூடியதாகவும், ஈர்க்கக்கூடியதாகவும்** மாற்றுவதற்கு எழுத்துக்களை வரிசைப்படுத்தும் கலை மற்றும் நுட்பமாகும். நம்முடையது போன்ற ஒரு தொழில்நுட்ப வடிவமைப்பு அமைப்பில், எழுத்துரு அமைப்பு மூன்று முக்கிய நோக்கங்களுக்காகப் பயன்படுகிறது:

1. **தெளிவு (Clarity):** சிக்கலான குறியீடுகள் (code) மற்றும் கணிதங்களை எளிதாக வேறுபடுத்திப் பார்ப்பதை உறுதி செய்தல்.
2. **வரிசைமுறை (Hierarchy):** மிக முக்கியமான தலைப்புகளிலிருந்து சிறிய விவரங்கள் வரை வாசகரின் பார்வையை வழிநடத்துதல்.
3. **அணுகல்தன்மை (Accessibility):** சாதனம் அல்லது பார்வைத்திறன் பாகுபாடின்றி அனைவரும் உரையை வாசிக்க முடிவதை உறுதி செய்தல்.

---

## இது ஏன் முக்கியமானது?

* **குறைக்கப்பட்ட அறிவாற்றல் சுமை (Reduced Cognitive Load):** சரியான இடைவெளி மற்றும் எழுத்துரு தேர்வுகள் மாணவர்கள் சோர்வின்றி விரைவாகக் கற்க உதவுகின்றன.
* **தொழில்முறை நம்பிக்கை (Professional Trust):** சீரான எழுத்துரு நடை, நமது கல்விக்கருவிகளை நம்பகமானதாகவும் நன்கு ஆராய்ச்சி செய்யப்பட்டதாகவும் மாற்றுகிறது.
* **சூழல் விழிப்புணர்வு (Context Awareness):** "எச்சரிக்கை", "குறியீட்டு துணுக்கு" மற்றும் "பொதுவான அறிவுறுத்தல்" ஆகியவற்றிற்கு இடையிலான வேறுபாட்டை பயனர்கள் உடனடியாக அறிய இது உதவுகிறது.

## பயன்பாட்டு உதாரணங்கள் (Markdown தொடரியல்)

உள்ளடக்கத்தை உருவாக்குபவராக, நீங்கள் CSS எழுத வேண்டிய அவசியமில்லை. நீங்கள் **Markdown** பயன்படுத்தினால் போதும், குருகுலம் வடிவமைப்பு அமைப்பு தானாகவே அதன் பாணிகளை (styling) கவனித்துக் கொள்ளும்.

### 1. தலைப்புகளுடன் கூடிய கட்டமைப்பு
வரிசைமுறையை உருவாக்க ஹேஷ்டேக்குகளை (#) பயன்படுத்தவும். ஒரு நிலையைத் தவிர்க்க வேண்டாம் (உதாரணமாக, `#` இலிருந்து நேரடியாக `###` இற்குச் செல்ல வேண்டாம்).

```markdown
# நிலை 1 தலைப்பு (பக்கத்தின் தலைப்பு)
## நிலை 2 தலைப்பு (முக்கியப் பிரிவு)
### நிலை 3 தலைப்பு (உட்பிரிவு)
```

### 2. முக்கியத்துவம் மற்றும் கவனம்
ஒரு பத்திக்குள் இருக்கும் முக்கிய கருத்துக்களை முன்னிலைப்படுத்த இவற்றைப் பயன்படுத்தவும்.

```markdown
முக்கியமான சொற்கள் அல்லது செயல்களுக்கு **தடிமனான உரையை (bold)** பயன்படுத்தவும்.
மேற்கோள்கள், வரையறைகள் அல்லது முக்கியத்துவத்திற்கு *சாய்வான உரையை (italic)* பயன்படுத்தவும்.
கைவிடப்பட்ட தகவல்களுக்கு ~~அடிக்கோடிட்ட உரையை (strikethrough)~~ பயன்படுத்தவும்.
```

### 3. வாசிப்புத் திறனுக்கான பட்டியல்கள்
படிகள் அல்லது அம்சங்களை விளக்குவதற்கு நீண்ட பத்திகளை விட பட்டியல்கள் சிறந்தவை.

```markdown
- வரிசைப்படுத்தப்படாத புள்ளிகளுக்கு புல்லட்ஸ்கள்.
- தகவல்களை எளிதாக ஸ்கேன் செய்ய உதவுகிறது.

1. வரிசைமுறைப் படிகளுக்கு எண்கள்.
2. அறிவுறுத்தல்களைப் பின்பற்றுவதற்கு உதவுகிறது.
```

### 4. தொழில்நுட்ப மற்றும் இன்லைன் குறியீடு (Code)
சாதாரண சொற்களுக்கும் தொழில்நுட்ப பெயர்களுக்கும் (மாறிகள் அல்லது கோப்பு பாதைகள் போன்றவை) இடையே உள்ள வேறுபாட்டைக் காட்டவும்.

```markdown
தீமை நிறுவ, `hugo server` என்பதை இயக்கவும். 
உள்ளமைவு (Configuration) `hugo.toml` கோப்பில் சேமிக்கப்பட்டுள்ளது.
```

### 5. அர்த்தமுள்ள மேற்கோள் தொகுதிகள் (Blockquotes)
"ஆலோசனைகள்", "குறிப்புகள்" அல்லது "எச்சரிக்கைகளுக்கு" இவற்றைப் பயன்படுத்தவும். உடல் உரையிலிருந்து இவை தனித்துத் தெரியும்படி நமது அமைப்பு வடிவமைக்கப்பட்டுள்ளது.

```markdown
> **குறிப்பு:** இணையத்தில் எழுத்துரு அமைப்பு, அதிகபட்ச வாசிப்புத் திறனுக்காக 1.5 முதல் 1.6 வரையிலான வரி உயரத்தைக் (line height) கொண்டிருக்க வேண்டும்.
```

## குருகுலத்திற்கான சிறந்த நடைமுறைகள்
* **பத்திகளைச் சுருக்கமாக வைக்கவும்:** இணைய வாசிப்பிற்கு 3-4 வாக்கியங்கள் போதுமானவை.
* **விளக்கமான இணைப்புகளைப் பயன்படுத்தவும்:** [இங்கே கிளிக் செய்யவும்](url) என்பதற்குப் பதிலாக [எழுத்துரு வழிகாட்டியைப் படிக்கவும்](url) என்பதைப் பயன்படுத்தவும்.
* **அனைத்து எழுத்துக்களையும் பெரிய எழுத்துக்களாக (All-Caps) எழுதுவதைத் தவிர்க்கவும்:** இது வாசிப்பதற்கு கடினமாக இருக்கும் மற்றும் கல்விச் சூழலில் "கத்துவது" போன்ற உணர்வைத் தரும்.
