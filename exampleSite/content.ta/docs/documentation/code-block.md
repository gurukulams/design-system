தொழில்நுட்ப ஆவணமாக்கலில், குறியீடுகளை (code) தெளிவாகக் காண்பிப்பது மிகவும் முக்கியம். **குருகுலம் வடிவமைப்பு அமைப்பு (Gurukulams Design System)** புரோகிராமிங் ஸ்கிரிப்டுகள், உள்ளமைவு கோப்புகள் (configuration files) மற்றும் டெர்மினல் கட்டளைகளை எளிதாக வாசிக்கவும் புரிந்துகொள்ளவும் "Syntax Highlighting" (குறியீடு வண்ணமயமாக்கல்) முறையைப் பயன்படுத்துகிறது.

## குறியீட்டுத் தொகுதிகளை (Code Blocks) எழுதும் முறை

Markdown முறையில், ஒரு வாக்கியத்தின் உள்ளே வரும் குறியீட்டிற்கும், தனித் தொகுதியாக வரும் குறியீட்டிற்கும் வித்தியாசம் உள்ளது.

### 1. வரிக்குள் அமையும் குறியீடு (Inline Code)
ஒரு பத்திக்கு உள்ளேயே ஒரு சிறிய குறியீடு, கோப்பின் பெயர் அல்லது மாறியின் பெயரை (variable name) முன்னிலைப்படுத்த ஒற்றை பேக்டிக்ஸ் (backticks) `` ` `` பயன்படுத்தவும்.

**உதாரணம்:**
சர்வரைத் தொடங்க, உங்கள் டெர்மினலில் `hugo server` என்ற கட்டளையைப் பயன்படுத்தவும்.

## ஆதரிக்கப்படும் மொழிகள்

நமது திட்டப்பணிகளில் பொதுவாகப் பயன்படுத்தப்படும் பல்வேறு மொழிகளை நமது அமைப்பு ஆதரிக்கிறது:
* **Web:** `html`, `css`, `javascript`, `typescript`
* **Backend:** `java`, `python`, `sql`
* **Data:** `json`, `yaml`, `xml`
* **Terminal:** `bash` அல்லது `shell`

## குறியீட்டுத் தொகுதிகளை ஏன் பயன்படுத்த வேண்டும்?

* **வாசிப்புத் திறன்:** வண்ணங்கள் இருப்பதால் மாணவர்கள் செயல்பாடுகள் (functions), மாறிகள் (variables) மற்றும் குறிப்புகளை (comments) எளிதில் வேறுபடுத்திப் பார்க்க முடியும்.
* **நகலெடுக்க எளிதானது:** குறியீட்டுத் தொகுதிகள் "Monospaced" எழுத்துருவைப் பயன்படுத்துகின்றன. இதில் ஒவ்வொரு எழுத்தும் ஒரே அகலத்தைக் கொண்டிருப்பதால், குறியீட்டின் வரிசை மாறாமல் அப்படியே இருக்கும்.
* **தெளிவான பிரிப்பு:** இது பொதுவான விளக்க உரையிலிருந்து தொழில்நுட்ப அறிவுறுத்தல்களைத் தனியாகப் பிரித்துக் காட்டுகிறது.

## உள்ளடக்கத்தை உருவாக்குபவர்களுக்கான சிறந்த நடைமுறைகள்

* **எப்போதும் மொழியைக் குறிப்பிடவும்:** வெறும் ` ``` ` என்று எழுதுவதை விட ` ```java ` என்று எழுதுவது சிறந்தது. இது கணினிக்கு எந்த வண்ணத்தைப் பயன்படுத்த வேண்டும் என்று சரியாகத் தெரிவிக்கும்.
* **மிக நீண்ட வரிகளைத் தவிர்க்கவும்:** உங்கள் குறியீடு மிகவும் நீளமாக இருந்தால், அதை அடுத்தடுத்த வரிகளுக்கு மாற்றவும். இது மொபைலில் பார்ப்பவர்களுக்கு வசதியாக இருக்கும்.
* **குறிப்புகளைப் (Comments) பயன்படுத்தவும்:** குறியீட்டிற்குள் அந்தந்த வரிகள் என்ன செய்கின்றன என்பதை விளக்க குறிப்புகளைச் சேர்க்கவும் (உதாரணமாக, ஜாவாவில் `// இது ஒரு குறிப்பு`).

> **குறிப்பு:** டெர்மினல் கட்டளைகளுக்கு `bash` என்ற குறிச்சொல்லைப் பயன்படுத்தவும். இது அந்தக் கட்டளையை கமாண்ட் பிராம்ட்டில் (command prompt) தட்டச்சு செய்ய வேண்டும் என்பதை மாணவர்கள் புரிந்துகொள்ள உதவும்.
